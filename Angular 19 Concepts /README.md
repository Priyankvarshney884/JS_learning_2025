# Angular 19 Concepts - Memory-First System

Use this folder like a **learning OS**:
- Every concept gets one note.
- Every note must include **proof** (mini code/demo/test/screenshot/log).
- Revise with quick flash prompts and weekly review.

## Interview-Day Shortcut

- Open `Interview-Prep/Senior-Fullstack-Interview-Day-Guide.md` for a consolidated prep plan tailored to Angular + Java + Spring fullstack roles.

## How To Add a New Concept (5 mins)

1. Copy `_templates/concept-template.md` into the right topic folder.
2. Rename it: `NN-concept-name.md` (example: `03-signals-vs-observable.md`).
3. Fill the sections quickly.
4. Add at least 1 item in `proofs/` and link it in the note.

## "Proof" Rule (Very Important)

For each concept, attach at least one:
- tiny runnable code snippet
- bug + fix write-up
- screenshot of output
- test case
- short voice/video explanation link

If there is no proof, treat concept as "not learned yet".

## Revision Rhythm (Efficient Memory)

- Day 0: Learn + write concept note + add proof
- Day 1: 5-minute recall from prompts (without seeing note)
- Day 3: Re-attempt same proof from scratch
- Day 7: Teach concept in 60 seconds (record/self-explain)
- Day 14: Solve one interview-style question from concept

## Concept Quality Checklist

Before marking done, confirm:
- I can explain this concept in simple words.
- I know one real use-case and one anti-pattern.
- I can write a minimal code example without copy-paste.
- I linked at least one proof artifact.
